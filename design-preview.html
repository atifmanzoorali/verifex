05 — Brand Voice
UI Copy — Do vs. Don't
Do
"You don't have any API keys yet. Create one to get started."
Don't
"No API keys have been created by the user at this time."
Do
"This email has no mail server. It will bounce."
Don't
"Invalid MX record detected. DNS lookup returned NXDOMAIN."
Do
"Delete this key? This can't be undone."
Don't
"Are you absolutely sure you want to permanently delete this key?"
Voice Rules
Contractions
Always — "don't", "won't", "you've"
User address
Second person — "your", "you"
Emojis
Never
Hashtags
Never
Caption length
1–3 lines maximum
Passive voice
Avoid always
README Exception
The README uses a warmer, more welcoming tone. Same clarity, no fluff — but it invites contribution rather than instructing a user. Explicitly welcomes first-time contributors.

16 — Token Architecture
Tier 1
Global Tokens
Raw primitive values. Never used directly in components. The single source of truth for every value in the system.
--color-teal-600: #1A7A78
--font-sans: 'Inter'
--size-4: 16px
--radius-sharp: 2px
--duration-base: 200ms
Tier 2
Semantic Tokens
Role assignments. Maps global values to their purpose. Used everywhere across the product UI.
--primary: var(--color-teal-600)
--background: var(--color-cream-100)
--text-primary: var(--color-ink)
--border: #D9D3C5
--radius: var(--radius-sharp)
Tier 3
Component Tokens
Component-specific overrides. Allows a single component to change without affecting the semantic layer.
--button-primary-bg: var(--primary)
--input-border-focus: var(--primary)
--card-padding: var(--size-6)
--nav-active-bg: #EAF3F3
--button-radius: var(--radius)
The Rule — Which Tier to Change
Brand colour change
Edit Tier 1 → everything updates automatically
Change what "primary" means
Edit Tier 2 → all components using primary update
Change one button's style
Edit Tier 3 → only that component changes

22 — Link Styles

Verifex validates email addresses in two steps. First it checks the format — see the format validation rules for the full spec. Then it performs an MX record lookup using the DNS resolution module. If either check fails, the email is marked invalid. You can read more about how this works in the architecture documentation.

Links always use text-decoration: underline. The underline is the signal. Never rely on colour alone to indicate a link — that fails accessibility requirements.

25 — Dark Mode Decision

Verifex does not support dark mode.
This is a decision, not an oversight.

Why no dark mode

Swiss International Typographic Style is rooted in print. The warmth of #F0ECE3 and the contrast of black type on a light field is inseparable from this aesthetic. A dark version would be a different design language — not a translation.

The standard we hold

A single, immaculately executed light theme is more impressive to a senior engineer evaluating this work than two themes executed to half the standard. Quality over coverage.
